Para mantener la integridad de los datos siguiendo tus estándares previos.


Type Safety: Cada tabla en Supabase debe tener una interfaz TypeScript correspondiente en app/interfaces/. No usar any.


Naming: Tablas en snake_case, pero las interfaces y objetos en el código en camelCase.


Manejo de Errores: Usar excepciones directas en lugar de valores mágicos como null.
